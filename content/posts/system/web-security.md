## 常见网络攻击类型

| 攻击类型              | 攻击危害                                        | 防护方案                                           |
| :-------------------- | :---------------------------------------------- | :------------------------------------------------- |
| SQL 注入              | 插入恶意 SQL 语句，实现拖库、删库、数据篡改     | 1. 使用参数化查询 / 预编译语句，禁止字符串拼接 SQL |
|                       |                                                 | 2. 数据库账号权限最小化                            |
|                       |                                                 | 3. 部署 WAF 拦截注入特征                           |
| XSS 跨站脚本攻击      | 注入恶意 JS，窃取 Cookie / 会话、钓鱼、劫持账号 | 1. 转义用户输入特殊字符                            |
|                       |                                                 | 2. Cookie 设置 HttpOnly                            |
|                       |                                                 | 3. 启用 CSP 内容安全策略                           |
| CSRF 跨站请求伪造     | 利用登录会话，未授权执行改密、转账、删帖等操作  | 1. 关键接口增加随机 Token 校验                     |
|                       |                                                 | 2. 重要操作二次验证                                |
|                       |                                                 | 3. 校验 Referer/Origin 来源                        |

|                       |                                                 | 3. 启用验证码 / 人机验证                           |
| 文件上传漏洞          | 上传木马后门，获取服务器控制权，删站、挖矿      | 1. 严格校验文件后缀、类型、内容                    |

|                       |                                                 | 3. 重命名文件，独立存储                            |
| 暴力破解 & 撞库       | 批量试密，窃取后台 / 账号管理权限               | 1. 登录限流、失败锁定，增加验证码                  |
|                       |                                                 | 2. 启用双因素认证 2FA                              |
|                       |                                                 | 3. 禁用弱密码，定期改密                            |
| SSRF 服务器端请求伪造 | 操控服务器访问内网，窃取配置、攻击内网资产      | 1. 禁止请求内网 IP，限制域名白名单                 |
|                       |                                                 | 2. 禁用高危请求函数                                |
|                       |                                                 | 3. 容器化隔离权限                                  |
| 命令注入              | 插入系统命令，删文件、挖矿、反弹木马            | 1. 禁用系统命令执行函数                            |
|                       |                                                 | 2. 参数严格过滤校验                                |
|                       |                                                 | 3. 最小化服务运行权限                              |
| 目录遍历 / 路径穿越   | 构造../路径，读取网站外配置、源码、密码文件     | 1. 路径参数白名单校验                              |
|                       |                                                 | 2. 禁止../等特殊字符                               |
|                       |                                                 | 3. 严格控制目录访问权限                            |
| 恶意爬虫 & 薅羊毛     | 批量爬取数据、刷接口，耗资源、扰业务            | 1. 接口加签名、频率限制                            |
|                       |                                                 | 2. UA 校验 + 行为风控                              |
|                       |                                                 | 3. 隐藏敏感接口                                    |

### 进阶思路

不是“封人”，而是：降低攻击成功率 + 提高攻击成本

```txt
用户访问
↓
Cloudflare（CDN + WAF + 速率限制 + DDoS 清洗 + 隐藏源IP）
↓
Nginx（仅允许 Cloudflare IP 访问 + 多网站分发 + 二次安全拦截）
↓
后端服务

- site1: 127.0.0.1:8001
- site2: 127.0.0.1:8002
- site3: 127.0.0.1:8003
- site4: 127.0.0.1:8004
- site5: 127.0.0.1:8005
```

监控：Cloudflare Dashboard 查看攻击日志 + Nginx access.log + 应用日志

- Cloudflare: CDN + WAF + Rate Limit
- 路径穿越（必须修）严格控制目录访问权限
- 配置自动备份（每天备份到异地）

## Cloudflare（CDN + WAF + Rate Limit + DDoS防护）

Cloudflare 隐藏你的真实服务器IP
Cloudflare 必须有域名, 把域名解析到 CDN 节点. 开启 CDN 代理（橙色云朵）
服务器防火墙：只允许 CDN 节点 IP 访问，拒绝所有其他 IP

假设你的域名：xxx.com5 个网站用子域名区分：
site1.xxx.com → 指向你的服务器 IP
site2.xxx.com → 同上

DNS & 代理模式：所有域名 DNS 都走 Cloudflare（橙云标），隐藏真实服务器 IP

Cloudflare：DDoS缓解、全球CDN、WAF规则、Rate Limit（免费版即可）

关键：Nginx 只监听 127.0.0.1 或 Unix Socket，公网 80/443 端口完全由 Cloudflare 接管，防止直接访问绕过 WAF。

### Django 生产环境关键设置

在 settings.py 中，必须修改以下配置以保证安全：

CSRF Token 防止跨站请求伪造 中
关闭错误回显 生产环境不显示报错信息，防止信息泄露 低
DEBUG = False: 绝对不能在线上开启调试模式，这会暴露大量敏感信息。

cors
ALLOWED_HOSTS: 必须设置，指定允许访问的域名或 IP，防止 HTTP Host 头攻击。

SECRET_KEY: 从环境变量或文件中读取，绝不能硬编码在代码里。

## 攻击模式分析：

- 环境配置文件：/.env、/app/.env、/config/.env 等
- 配置文件：/config.js、/settings.json、/secrets.json 等
- 数据库配置：/wp-config.php.\* 系列文件
- 应用文件：/app.js、/main.js、/server.js 等
- 路径遍历攻击：/@fs/../../../../../root/.env

## 安全建议：

- 检查服务器防火墙规则，限制不必要的端口访问
- 配置Web服务器，阻止对敏感文件的直接访问
- 监控异常IP地址的访问行为

第一阶段：紧急修复（立即执行）
