## 常见网络攻击类型

| 攻击类型              | 攻击危害                                        | 防护方案                                           |
| :-------------------- | :---------------------------------------------- | :------------------------------------------------- |
| SQL 注入              | 插入恶意 SQL 语句，实现拖库、删库、数据篡改     | 1. 使用参数化查询 / 预编译语句，禁止字符串拼接 SQL |
|                       |                                                 | 2. 数据库账号权限最小化                            |
|                       |                                                 | 3. 部署 WAF 拦截注入特征                           |
| XSS 跨站脚本攻击      | 注入恶意 JS，窃取 Cookie / 会话、钓鱼、劫持账号 | 1. 转义用户输入特殊字符                            |
|                       |                                                 | 2. Cookie 设置 HttpOnly                            |
|                       |                                                 | 3. 启用 CSP 内容安全策略                           |
| CSRF 跨站请求伪造     | 利用登录会话，未授权执行改密、转账、删帖等操作  | 1. 关键接口增加随机 Token 校验                     |
|                       |                                                 | 2. 重要操作二次验证                                |
|                       |                                                 | 3. 校验 Referer/Origin 来源                        |
| DDoS / CC 攻击        | 打满带宽、耗尽服务器资源，导致网站瘫痪          | 1. 接入高防 IP/CDN 清洗流量                        |
|                       |                                                 | 2. 配置限流、IP 黑名单                             |
|                       |                                                 | 3. 启用验证码 / 人机验证                           |
| 文件上传漏洞          | 上传木马后门，获取服务器控制权，删站、挖矿      | 1. 严格校验文件后缀、类型、内容                    |
|                       |                                                 | 2. 上传目录禁脚本执行权限                          |
|                       |                                                 | 3. 重命名文件，独立存储                            |
| 暴力破解 & 撞库       | 批量试密，窃取后台 / 账号管理权限               | 1. 登录限流、失败锁定，增加验证码                  |
|                       |                                                 | 2. 启用双因素认证 2FA                              |
|                       |                                                 | 3. 禁用弱密码，定期改密                            |
| SSRF 服务器端请求伪造 | 操控服务器访问内网，窃取配置、攻击内网资产      | 1. 禁止请求内网 IP，限制域名白名单                 |
|                       |                                                 | 2. 禁用高危请求函数                                |
|                       |                                                 | 3. 容器化隔离权限                                  |
| 命令注入              | 插入系统命令，删文件、挖矿、反弹木马            | 1. 禁用系统命令执行函数                            |
|                       |                                                 | 2. 参数严格过滤校验                                |
|                       |                                                 | 3. 最小化服务运行权限                              |
| 目录遍历 / 路径穿越   | 构造../路径，读取网站外配置、源码、密码文件     | 1. 路径参数白名单校验                              |
|                       |                                                 | 2. 禁止../等特殊字符                               |
|                       |                                                 | 3. 严格控制目录访问权限                            |
| 恶意爬虫 & 薅羊毛     | 批量爬取数据、刷接口，耗资源、扰业务            | 1. 接口加签名、频率限制                            |
|                       |                                                 | 2. UA 校验 + 行为风控                              |
|                       |                                                 | 3. 隐藏敏感接口                                    |

## 攻击模式分析：

- 环境配置文件：/.env、/app/.env、/config/.env 等
- 配置文件：/config.js、/settings.json、/secrets.json 等
- 数据库配置：/wp-config.php.\* 系列文件
- 应用文件：/app.js、/main.js、/server.js 等
- 路径遍历攻击：/@fs/../../../../../root/.env

## 安全建议：

- 检查服务器防火墙规则，限制不必要的端口访问
- 配置Web服务器，阻止对敏感文件的直接访问
- 监控异常IP地址的访问行为

第一阶段：紧急修复（立即执行）

- 禁用调试模式 - 防止信息泄露
- 配置ALLOWED_HOSTS - 限制可访问域名
- 完善CSRF防护 - 增强跨站请求安全

第二阶段：重要改进（今日内完成）

- 加强密码策略 - 提升账户安全
- 配置CORS白名单 - 控制跨域访问
